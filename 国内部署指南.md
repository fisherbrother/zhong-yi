# 中医数据库 - 中国大陆可访问部署指南

## 🚀 快速部署选项

### 方案一：GitHub Pages（推荐）
**优点**：免费、稳定、中国大陆可访问
**缺点**：需要GitHub账号

#### 部署步骤：
1. **创建GitHub仓库**
   - 登录GitHub
   - 创建新仓库，命名为 `tcm-database` 或其他名称
   - 不要初始化README

2. **推送代码到GitHub**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/你的用户名/你的仓库名.git
   git push -u origin main
   ```

3. **启用GitHub Pages**
   - 进入仓库Settings
   - 找到Pages设置
   - Source选择"Deploy from a branch"
   - Branch选择"gh-pages" / "(root)"
   - 保存

4. **自动部署**
   - 项目已配置GitHub Actions工作流
   - 推送代码到main分支会自动触发部署
   - 访问地址：`https://你的用户名.github.io/你的仓库名`

### 方案二：Gitee Pages（国内首选）
**优点**：国内服务，访问速度更快
**缺点**：需要实名认证

#### 部署步骤：
1. **注册Gitee账号并完成实名认证**
2. **创建Gitee仓库**
3. **推送代码到Gitee**
   ```bash
   git remote add gitee https://gitee.com/你的用户名/你的仓库名.git
   git push gitee main
   ```
4. **启用Gitee Pages服务**

### 方案三：国内云服务商
**阿里云OSS + CDN部署**

#### 部署步骤：
1. **购买阿里云OSS存储包**
2. **创建OSS存储桶**
3. **配置静态网站托管**
4. **上传构建文件**
5. **配置CDN加速**

## 📁 项目构建

项目已经配置好GitHub Pages构建：
```bash
npm run build:gh-pages
```

## 🔧 配置文件说明

- `.github/workflows/deploy.yml` - GitHub Actions自动部署配置
- `vite.config.gh-pages.ts` - GitHub Pages专用构建配置
- `dist/404.html` - SPA路由处理文件

## 🌐 访问地址

部署成功后，您的网站将可通过以下地址访问：
- GitHub Pages: `https://[用户名].github.io/[仓库名]`
- 自定义域名: 在仓库设置中配置CNAME

## ⚠️ 注意事项

1. **数据库连接**：确保Supabase配置正确
2. **环境变量**：在GitHub仓库设置中添加必要的Secrets
3. **备案问题**：
   - GitHub Pages不需要备案
   - 国内云服务商需要备案
4. **访问速度**：
   - GitHub Pages：中等速度
   - Gitee Pages：国内速度快
   - 阿里云+CDN：最优速度

## 🆘 常见问题

### Q: 页面空白怎么办？
A: 检查浏览器控制台错误，确认API配置正确

### Q: 路由跳转失败？
A: 已配置404.html处理SPA路由，确保文件存在

### Q: 构建失败？
A: 检查依赖包是否完整，运行`npm install`
